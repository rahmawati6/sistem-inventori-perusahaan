# Kebijakan Keamanan

## Versi yang Didukung

Perbaikan keamanan diterapkan pada versi terbaru di branch `main`.

## Melaporkan Kerentanan

Jangan melaporkan kerentanan keamanan melalui issue publik.

Gunakan fitur **Report a vulnerability** pada tab **Security** repository GitHub untuk mengirim laporan secara privat. Sertakan:

- Deskripsi dan dampak kerentanan
- Langkah untuk mereproduksi masalah
- Versi, commit, atau lingkungan yang terdampak
- Bukti konsep, log, atau screenshot yang sudah disamarkan
- Saran mitigasi jika tersedia

Maintainer akan berusaha mengonfirmasi penerimaan laporan dalam tujuh hari dan memberikan pembaruan selama proses investigasi. Waktu perbaikan bergantung pada tingkat keparahan dan kompleksitas masalah.

Hindari mengakses, mengubah, atau menghapus data milik pihak lain selama melakukan pengujian keamanan.
